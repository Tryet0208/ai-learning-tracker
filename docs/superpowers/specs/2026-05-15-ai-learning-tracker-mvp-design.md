# AI学习追踪器 MVP 设计文档

> 2026-05-15 | v1.0

## 产品定位

帮 AI 小白从不入门到能独立用 AI 工具、部署项目的学习平台。

**用户成长路径：** 小白（不会用AI） → 入门（会用AI工具） → 落地（能独立部署项目）

**目标用户：** 国内大学生/职场新人，无外网环境，预算有限，想学 AI 实战技能。

## 核心功能

### 1. 首页仪表盘

- 今日待完成任务列表（从课程模块自动生成）
- 连续打卡天数 + 本周统计
- 等级进度条（入门 → 进阶 → 独立）
- 快捷入口：继续学习、项目工坊

### 2. 学习中心

- **课程路线**：按「入门→实战→独立」三阶段组织
  - 阶段一：会用 AI 对话、Prompt 工程、环境搭建
  - 阶段二：Claude Code/Cursor 使用、Python 脚本、本地部署
  - 阶段三：理解项目结构、改配置、独立落地
- **技能模块**：每个模块 = 学（概念讲解）→ 练（课后练习）→ 测（小测验）
- **内容内置**：学习内容由 AI 生成，直接在页面内展示，无需跳转外链
- **固定地址**：每个模块有永久 URL（如 `/learn/week/1/day/1`），已完成模块可随时回看复习
- **外部链接降级为延伸阅读**：在内容底部附上参考链接，用户可选是否点击
- **AI 问答**：学习过程中随时提问（MVP 第二阶段实现）

### 3. 内置内容系统

**内容生成流程：**
```
curriculum.json（课程大纲）
       ↓
   AI 生成模块内容（Markdown）
       ↓
   存储到 Module 表（content 字段）
       ↓
   前端渲染 Markdown + 附带延伸链接
       ↓
   每个模块固定 URL，永久可访问
```

**质量保证：**
- AI 根据 curriculum 中定义的 theme/goal/task 生成中文教学内容
- 生成的内容标记为 draft，需人工审核后切换为 published
- 支持重新生成某个模块的内容（覆盖旧版本）
- 内容包含：通俗解释、实战示例、关键概念、课后小测

**固定地址设计：**
- 每个模块 URL：`/learn/week/:week/day/:day`
- 已完成模块在课程列表中标记为"已学"，点击可回看
- 今日任务页面嵌入模块内容，同时提供"在新页面打开"按钮
- 课程目录页列出所有模块，不限当天，用户可自由浏览

### 3. 项目工坊

- 精选项目列表（平台内置 + GitHub/Gitee 双源）
- 「三步落地」体验：下载资源包 → 执行一键脚本 → 打开即用
- 保姆级部署指南：中文说明、常见报错处理、截图引导
- MVP 阶段至少 1 个完整项目

### 4. 打卡系统

- 每日学习打卡（完成至少一个任务即可打卡）
- 连续天数统计 + 最长纪录
- 打卡日历视图

### 5. PWA 移动端

- 添加到手机桌面，类 App 体验
- Service Worker 离线缓存核心页面
- 响应式 UI，手机/平板/桌面自适应

## 每日任务引擎（核心）

### 三阶段分层

| 阶段 | 标签 | 每日任务结构 | 任务侧重 |
|------|------|-------------|---------|
| 小白 | 刚接触 AI | 概念认知 ×1 + 简单体验 ×1 | 理解为主，上手为辅 |
| 入门 | 会用 AI 工具 | 工具实操 ×1 + 知识点 ×1 + 拓展练习 ×1 | 动手为主，理解跟进 |
| 落地 | 能独立部署 | 项目实战 ×1 + 自主探索 ×1 | 实战为主，自由探索 |

### 任务类型与完成标准

| 任务类型 | 说明 | 建议产出 |
|----------|------|---------|
| 概念认知 | 阅读/观看，理解基本概念 | 完成几道选择题 |
| 简单体验 | 打开 AI 工具试用 | 截图第一段对话 |
| 工具实操 | 跟着步骤操作工具 | 上传操作结果截图 |
| 知识点 | 掌握关键概念 | 用自己的话写一段总结 |
| 项目实战 | 落地真实项目 | 提交项目运行截图 |
| 自主探索 | 找资料深挖方向 | 写一段学习笔记 |

- 产出仅为建议，**不做强制要求**，用户可跳过任意任务
- 必做/选做标签仅为引导，不影响打卡
- 完成至少一个任务即可打卡

### 任务生成规则

- 根据 `user.current_level`（小白/入门/落地）匹配对应阶段的任务模板
- 每个任务包含：标题、描述（学什么/能做什么/预计时长）、资源链接、任务类型
- 现有 `curriculum.json`（12 周课程）作为内容池，按阶段和进度取用
- 同一主题在不同阶段自动变换类型：小白→概念认知，入门→工具实操，落地→项目实战

## 数据模型

### 现有表（无需改动）
- **User** — 用户信息、等级、学习时间、连续天数
- **LearningTask** — 学习任务（标题、类型、时长、状态），新增 `module_id` 外键
- **Resource** — 学习资源（文章/视频，标签，难度）
- **CheckIn** — 每日打卡记录

### 新增表：Module

| 字段 | 类型 | 说明 |
|------|------|------|
| id | int (PK) | 主键 |
| week | int | 所属周数 |
| day | int | 所属天数 |
| title | str | 模块标题 |
| theme | str | 所属主题 |
| goal | str | 学习目标 |
| content | text (Markdown) | AI 生成的教学内容 |
| status | str | draft / published |
| external_links | text (JSON) | 延伸阅读链接列表 |
| created_at / updated_at | datetime | 时间戳 |

- 永久 URL 映射：`/learn/week/:week/day/:day` → Module 表查询
- 用户可自由浏览任意模块，不受当日任务限制

## 不做的事（MVP）

- 微信登录/小程序
- 付费/会员体系
- 社区/评论功能
- AI 实时问答
- 桌面客户端
- 多语言
- 复杂权限系统

## 验证指标

- 10 个真实用户能完成完整学习流程
- 移动端添加到桌面后操作流畅
- 至少 1 个项目被用户成功部署到本地
